# Project Management Guide

## Planning
- Define clear goals and scope
- Break into tasks (< 1 day each)
- Estimate effort for each task
- Identify dependencies
- Set milestones

## Execution
- Daily standups (what, blockers, plan)
- Track progress visually (kanban board)
- Communicate delays early
- Remove blockers fast

## Communication
- Status updates to stakeholders
- Technical decisions documented
- README for onboarding
- Changelog for releases

## Tools
- Issues for task tracking
- Labels for categorization (bug, feat, docs)
- Milestones for sprints/versions
- Projects for kanban boards
- Actions for CI/CD
